Windows Active Directory

Part 2 : Joining Linux clients to AD


CentOS 7 has made joining a Linux client to AD incredibly easy.  Get root access.  Install realmd and PackageKit.  Then type the following:
    nmcli c modify enp0s3 ipv4.dns [ip address of AD server]
    systemctl restart network
    realm discover harvey.virtnet
This should give you information about your domain and required packages.  If you get any errors then most likely DNS hasn't been configured properly and you'll need to troubleshoot that before continuing.
    realm join harvey.virtnet
Next you'll need to edit /etc/sssd/sssd.conf and edit line 16 to False to not use FQDN so you can omit the domain name for AD users.  Use systemctl to restart sssd.
If you don't receive any errors, you can go into your AD server and under Tools -> Active Directory Users and Computers -> Computers you should see the hostname of the machine you added.  Log in under an AD account or type id Administrator if you don't believe me.
