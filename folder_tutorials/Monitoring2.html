Adding Windows Server 2012 to Nagios


For this to work, we'll need the NSClient++ Agent installed on our Windows server.  Download the installer.

Run the installer and enter the ipaddress of your Nagios server, in my case by checking fw1 it's 192.168.2.45.  Enter the default ports, and click finish when done.

SSH into your nagios server and into /usr/local/nagios/etc/nagios.cfg and uncomment the line to allow nagios to parse all configurations in the servers folder.  Copy the windows.cfg file from objects into the servers folder and name it the hostname of the Windows Server.  Inside the ${hostname}.cfg file, change the ip address in define_host to the correct address, and change the hostname throughout the config by using search and replace :g/winserver/s//new_hostname/g in vi. 

At the bottom of this config file are the services that Nagios will be tracking.  You can (and should) write new ones.

When you're finished, run systemctl restart nagios and also enable it so it starts on boot.

Finally you might need to adjust the firewall with firewall-cmd --permanent --add-port=5666/tcp on both the host and agents.  Remember to reload the firewall when you're done!
