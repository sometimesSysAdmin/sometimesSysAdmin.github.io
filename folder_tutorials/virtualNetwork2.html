Create a Virtual Network

Part 2 : Setting up Kali and Metasploitable on your network

I like to keep a small virtual lab on my laptop so I can play around with it when time permits. For this tutorial you will need:

  • VirtualBox
  • pfSense iso
  • CentOS 7 Everything iso
  • Metasploitable
  • Kali Linux ova
Part 1 will be covering setting up pfSense to use as a router for your internal network.
Part 2 will cover installing Kali and Metasploitable
Part 3 will cover provisioning a PXE Boot Server
Part 4 will cover setting up Snort and Bro to look at your new network

So you have learned how to create VM's.  When you browse to the Metasploitable and Kali download pages, you'll notice an option to download VM's.  For instance browse to the Kali Linux link provided and scroll halfway down the page.  You will see an option to download Kali Virtual Images.  Click this and it will take you to a page with VM's.  Scroll down to the VirtualBox Images (note that they provide them for a few virtualization platforms).  Download the Kali Linux 64 Bit VBox in open virtualization format .ova or click here.

Now that this is downloaded put it in a folder to save it and go to your VirtualBox Manager.  Click File -> Import Appliance and browse to the Kali Linux .ova file.  Click Next and look at the default values it has.  Note that we'll need to change the Network Adapter to the internal network.  Click import and let it do it's thing.  When it's finished go to the settings and set the Network Adapter to internal network.

Next we'll be installing the Metasploitable machine.  After downloading and unzipping this you are left with a few files including a virtual machine disk format .vmdk.  To install this on VirtualBox click new.  Make a machine name Metasploitable with Type Linux 2.6/3.x/4.x (64-Bit) and 1 GB of RAM.  For the Hard Disk select Use an existing virtual hard disk file and browse to the .vmdk.  Browse to settings and get it on the internal network.

Now finally, we're going to set up some DHCP leases in the pfSense.  Go back to the pfSense webConfiguration at 192.168.2.1.  Go to Services -> DHCP Server.  Scroll to the bottom for DHCP Static Mappings for this Interface and click add.  We'll start with Kali.  Enter the desired ip address (use any address in 192.168.2.0/27 that isn't taken), the hostname (kali), and a short description.  Leave all other defaults and scroll to the bottom and click save.  Now repeat the process with Metasploitable (hostname is metasploitable).  Now you can run nslookup on metasploitable.your.domain.

In Part 3, we will provision a PXE Boot server to quickly create CentOS VM's for new services.
